GGF PID DECODER STUDIO
AI AGENT INSTALLATION HANDOFF

Purpose
-------
This file is for a user who wants an AI agent to install GGF PiD Decoder
Studio for them.

The agent is not creating an installer product.
The agent is installing the app on the user's machine.

When the work is finished, the user should have:

1. The app installed in a clean GGF folder.
2. A simple click-to-run BAT file.
3. A short START-HERE text file.
4. A verified local browser launch.


What This App Is
----------------
GGF PiD Decoder Studio is a local browser GUI for NVIDIA PiD.

In practical terms, this app behaves like an AI image upscaler/enhancer:

- The user provides an input image.
- PiD encodes and decodes it through the official Flux-compatible PiD path.
- The app produces a higher-resolution PiD output plus comparison images.

It is not a full text-to-image generator.


Important Instruction To The Agent
----------------------------------
Do not build an installer.
Do not package a new installer.
Do not create a complex deployment system.

Your job is to install the app for the user and leave them with a way to run it.


System Prerequisites
--------------------
Before installing, check for these system-level requirements:

- Windows 10/11
- NVIDIA GPU
- NVIDIA driver with CUDA-capable PyTorch support
- Python 3.10 or Python 3.11 available through the `py` launcher
- Git for Windows available on PATH
- Internet access

These are system prerequisites, not normal Python package dependencies.
Do not try to silently fix missing system prerequisites inside the app install.

If Python, Git, or NVIDIA/CUDA support is missing or clearly broken, stop and
tell the user:

    Your system is missing one or more required setup items for local AI apps.
    Please go to:

    https://setup.getgoingfast.pro

    Use the automatic setup there, or follow its manual setup instructions.
    After that finishes, run this AI-agent install again.

Only continue once those prerequisites are present.


Ask The User Where To Install
-----------------------------
Ask the user:

    Which drive do you want to install GGF PiD on?
    I recommend C: unless you want to use another drive with more free space.

If the user says "default", use C:.

Do not install into system folders such as:

- C:\Windows
- C:\Program Files
- C:\Program Files (x86)
- System32
- protected OS folders


GGF Install Layout
------------------
Install the app under the drive chosen by the user:

    <DRIVE>:\GGF\GGF-PiD\
        CLICK-ME-TO-RUN.bat
        START-HERE.txt
        repo\

The GitHub repository must be cloned into:

    <DRIVE>:\GGF\GGF-PiD\repo

Public repository:

    https://github.com/gjnave/GGF-PiD.git


Install Steps
-------------
Replace C: with the drive chosen by the user.

1. Create the folders:

    mkdir C:\GGF
    mkdir C:\GGF\GGF-PiD

2. Clone or update the app source:

    If C:\GGF\GGF-PiD\repo\.git exists:

        git -C C:\GGF\GGF-PiD\repo pull --ff-only

    Otherwise:

        git clone https://github.com/gjnave/GGF-PiD.git C:\GGF\GGF-PiD\repo

3. Run the app's repo installer:

        C:\GGF\GGF-PiD\repo\Install-PiD.bat

   This creates the venv, installs Python packages, installs CUDA PyTorch,
   downloads/reuses PiD model files, and prepares the local app cache.

4. Create this run file:

    C:\GGF\GGF-PiD\CLICK-ME-TO-RUN.bat

   Exact content:

        @echo off
        setlocal
        cd /d "%~dp0repo"
        call "%~dp0repo\run.bat"

5. Create this short user instruction file:

    C:\GGF\GGF-PiD\START-HERE.txt

   Exact content:

        GGF PiD Decoder Studio

        To start the app:
        Double-click CLICK-ME-TO-RUN.bat

        Installed app folder:
        C:\GGF\GGF-PiD

        Installed repo:
        C:\GGF\GGF-PiD\repo

        Public repo:
        https://github.com/gjnave/GGF-PiD

        The app opens in your browser. It starts at port 7868. If that port is
        busy, it automatically tries the next available port.


Smoke Test
----------
After installation, verify the app starts.

1. Run:

    C:\GGF\GGF-PiD\CLICK-ME-TO-RUN.bat

2. Confirm a local browser app opens.

Expected URL:

    http://127.0.0.1:7868

If that port is busy, the app may use:

    http://127.0.0.1:7869
    http://127.0.0.1:7870

3. Confirm the page shows:

    PiD Decoder Studio

4. Confirm the status panel reports:

    GPU: an NVIDIA GPU / CUDA
    Checkpoint assets: present
    Gemma caption cache: present

5. Optional deeper test:
   Upload a small image, keep the default 2K Decode preset, and run Decode
   with PiD.

Successful outputs are written to:

    C:\GGF\GGF-PiD\repo\outputs


Troubleshooting
---------------
If Git, Python, or CUDA/NVIDIA support is missing:

Stop and send the user to:

    https://setup.getgoingfast.pro

Tell them to use the automatic setup or manual setup instructions there, then
rerun this install after their system is ready.

If model download fails:

Run this again:

    C:\GGF\GGF-PiD\repo\Install-PiD.bat

The app uses resumable curl downloads for the large PiD checkpoint files and
checks expected file sizes.

If port 7868 is busy:

No action is usually needed. The app automatically tries the next open port.
